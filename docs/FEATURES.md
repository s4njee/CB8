# CB8 — Features

A snapshot of what ships today and what's on the backlog. Generated by walking the codebase, not the README; if something isn't listed under **Implemented** it isn't in the build. Each implemented feature carries a sublist of the files that own it so the docs and the source stay legible together.

> **PLAN10 complete:** `src/web/` is the only frontend; the former
> `src/renderer/` React app has been removed and Electron loads the SPA
> via the embedded HTTP server.

## Implemented

### Library / Catalog

- **SQLite library DB with versioned migrations**, WAL mode, `busy_timeout`, idempotent post-migration indexes.
  - `src/main/db/schema/create.ts` — base DDL for all tables (comics, users, sessions, accounts, libraries, folders, tags, app_meta, …).
  - `src/main/db/schema/migrations.ts` — `migrateSchema`, version detection, `ensurePostMigrationIndexes`, `ensureSearchIndex`.

  - `src/main/db/schema/repairs.ts` — one-shot data repairs (`backfillCompletedOnFinalPage`, etc.).
  - `src/main/libraryDatabase.ts` — facade that fans calls out to per-domain modules.
  - `src/main/db/{comics,libraries,folders,tags,users,progress,bookmarks,history,favorites,appMeta}.ts` — domain queries.
- **FTS5 full-text search** over title / file_path / series / author / summary, with prefix-matching and Unicode-aware tokenization (≤7 ms typical at 100 k rows).

  - `src/main/db/comics.ts` — `buildFtsQuery`, plus the search branch in `queryComics` and `queryComicsForUser`.

  - `src/main/index.ts` — startup error handling around `db.initialize()`.

## Planned / backlog

Ordered top-down by user impact and likelihood of being requested.

### Tier 1 — Foundational gaps

1. **Continuous-scroll reading mode** for manga / webtoons (vertical strip).
2. **Duplicate detection** — flag the same content added twice under different paths.
3. **Per-user reader preferences synced server-side** — font / size / direction / theme follow the user across devices instead of living in `localStorage`.
4. **Offline PWA** — service-worker cache so recently-read items keep working without the server.
5. **Cover regeneration** — re-extract thumbnails for a selection (covers existing items being rebuilt after a metadata change).
6. **Brightness / contrast / sepia overlay** in the comic reader (eye comfort, OLED).

### Tier 2 — High value, well-scoped

7. **Bulk metadata edit** — set series / tags / rating across a multi-select.
8. **Star ratings** (1–5) — column, filter, sort.
9. **Persistent zoom level per comic.**
10. **Page cropping / auto-trim** to strip white/black borders.
11. **Scan scheduling** — cron-style "rescan nightly" in the embedded server.
12. **Storage analytics** — breakdown by format / series / library, identify space hogs.
13. **Broken file detection** — flag CBZs that fail to extract during scan.
14. **Reading streak / stats** — pages-per-day chart, session history (the `reading_history` table is already populated).
15. **EPUB highlight + annotation** — saved passages with notes; export to Markdown.
16. **EPUB table of contents panel** as a sidebar.
17. **Smart collections** — saved filter presets ("unread · CBZ · added this month").
18. **Reading lists / queues** — ordered lists separate from collections.
19. **Multi-user invite links** — token-based instead of admin-creates-user.

### Tier 3 — Nice to have

20. **Random pick button** ("open a random unread from this view").
21. **"New additions" shelf** distinct from the date-sorted grid.
22. **Related-by-series sidebar** when a comic is open.
23. **Reading time estimate** — "~12 min left at your pace".
24. **EPUB custom fonts**, **line-height / margin** controls.
25. **EPUB "Chapter 4 of 11"** progress display alongside %.
26. **Double-page cover detection** — page 0 always single in spread mode.
27. **Fit-to-content zoom** — auto-pick fit-width vs fit-height per page.
28. **Auto-advance / slideshow** timer for fixed-layout comics.
29. **Hardware media keys / presentation clicker** support in the Electron reader.
30. **Touch Bar support** (Mac) for page slider and bookmark.
